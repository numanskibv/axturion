# Contributing to MATS

Thank you for your interest.

MATS prioritizes architectural integrity.

---

## Before Opening a PR

1. Open an issue.
2. Describe the problem clearly.
3. Wait for design alignment.

---

## PR Requirements

- Small, focused changes
- Service-level tests included
- No framework leakage into domain
- Respect workflow integrity rules

---

## Code Review

All PRs require review.
Architectural consistency is mandatory.
